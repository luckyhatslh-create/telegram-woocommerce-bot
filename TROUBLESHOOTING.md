# Устранение неполадок

## Проблема: "Object of type bytes is not JSON serializable"

**Симптомы:** Бот запускается, но при отправке фото выдает ошибку.

**Причина:** В `providers/anthropic.py` передавались сырые байты вместо base64-encoded строки.

**Решение:** Обновите до последней версии - исправление добавлено в commit `d0579fb`.

## Проблема: "num_inference_steps: Must be less than or equal to 4"

**Симптомы:** Anthropic работает, но Replicate выдает ошибку валидации.

**Причина:** Модель `flux-schnell` поддерживает максимум 4 шага inference, а в `.env` было `STEPS_PREVIEW=20`.

**Решение:**
1. Обновите `.env`:
   ```bash
   STEPS_PREVIEW=4
   STEPS_HQ=4
   ```
2. Или используйте модель `flux-dev` если нужно больше шагов (платная).

## Проблема: Бот использует старую модель Anthropic

**Симптомы:** В `.env` указана правильная модель, но используется старая.

**Причина:** В оболочке (shell) экспортирована переменная `ANTHROPIC_MODEL`, которая переопределяет `.env`.

**Решение:**
1. Используйте скрипт `start_bot.sh` для запуска:
   ```bash
   ./start_bot.sh
   ```
2. Или вручную очистите переменные:
   ```bash
   unset ANTHROPIC_MODEL WC_KEY WC_SECRET
   python3 bot.py
   ```

## Проблема: "TELEGRAM_BOT_TOKEN не задан"

**Причина:** `load_dotenv()` вызывается после импорта `config`.

**Решение:** Обновите до последней версии - порядок импортов исправлен в `bot.py`.

## Проблема: "This event loop is already running" (Python 3.13)

**Причина:** Конфликт event loops в Python 3.13.

**Решение:** Обновите до последней версии - используется синхронный `main()` в commit `ac32316`.

## Проблема: "Request was throttled" (429 Rate Limit)

**Симптомы:** Replicate API возвращает ошибку 429 с сообщением о rate limiting.

**Причина:** При балансе < $5 на Replicate действует ограничение: 6 запросов в минуту с burst 1.

**Решение:**
1. Пополните баланс Replicate минимум на $5 для снятия ограничений
2. Или используйте встроенную retry логику (автоматически 3 попытки с задержкой 10 секунд)
3. Уменьшите частоту запросов от пользователей

**Обновление:**
- Добавлена автоматическая retry логика: 5 попыток с exponential backoff (15s, 30s, 45s, 60s)
- Обработка timeout errors при rate limiting
- Если после всех попыток все равно ошибка - пополните баланс Replicate

## Проблема: Инпейнтинг создает балаклаву вместо шапки

**Симптомы:** Маска накладывается на лицо, результат - балаклава/лыжная маска вместо обычной шапки.

**Причина:** Неправильное позиционирование маски инпейнтинга - слишком низко, захватывает глаза/нос/рот.

**Решение:**
1. Обновлена fallback маска в `utils/mask.py`:
   - Центр эллипса: ~0.25 * высоты (верхняя четверть)
   - Высота эллипса: ~0.35 * высоты
   - Нижняя граница ограничена 0.45 * высоты (не захватывает лицо)

2. Усилен промпт для FLUX fill:
   - Явный запрет на балаклавы/лицевые маски
   - Указание что шапка должна быть ТОЛЬКО на макушке
   - Запрет на покрытие глаз/носа/рта

3. Режим отладки `MASK_DEBUG=1`:
   - Включите в `.env`: `MASK_DEBUG=1`
   - Бот отправит дополнительное изображение с красной маской поверх портрета
   - Все промежуточные изображения сохраняются в `debug/`:
     * `{timestamp}_{id}_base.png` - сгенерированный портрет
     * `{timestamp}_{id}_mask.png` - маска в градациях серого
     * `{timestamp}_{id}_overlay.png` - портрет + красная маска (для визуализации)
     * `{timestamp}_{id}_result.png` - финальный результат после инпейнтинга

**Рекомендации:**
- При тестировании включите `MASK_DEBUG=1` чтобы видеть позицию маски
- Если маска все еще неправильная, отрегулируйте параметры в `utils/mask.py` функции `_ellipse_mask()`
